BrokerMAIL AI - Recovery System + Branding Improvements

STEP 1 — Server Startup Recovery System

The deferred retry loop is now working while the server remains online. We also need crash/restart recovery.

Requirements:

1. On API startup, scan all campaigns where:

   * status = sending
   * deferred queue items exist
   * lead status = sending
   * queue status = deferred

2. Automatically restart processing for eligible campaigns.

3. For deferred items where:
   retryAfter <= now

   immediately resume processing.

4. For campaigns where:

   * campaign.status = sending
   * activeJobs does not contain the campaign

   automatically recreate the processor.

5. If a lead is stuck in:
   sending

   while queue item is:
   deferred

   reset lead back to queued.

6. Add startup logs:

   [RECOVERY] Campaign found

   [RECOVERY] Deferred items found

   [RECOVERY] Processor restarted

Goal:

No campaign should require manual Resume after:

* server restart
* deployment
* Replit reboot
* process crash
